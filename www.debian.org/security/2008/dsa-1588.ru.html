DSA-1588-1 linux-2.6 -- отказ в обслуживании

Дата сообщения:
27.05.2008
Затронутые пакеты:
linux-2.6
Уязвим:
Да
Ссылки на базы данных по безопасности:
В каталоге Mitre CVE: CVE-2007-6712, CVE-2008-1615, CVE-2008-2136, CVE-2008-2137.
Более подробная информация:

В ядре Linux было обнаружено несколько уязвимостей, которые могут приводить к отказу в обслуживании. Проект Common Vulnerabilities and Exposures определяет следующие проблемы:

  • CVE-2007-6712

    Йоханнес Бауэр обнаружил переполнение целых чисел в подсистеме hrtimer на 64-битных системах. Эта уязвимость может использоваться локальными пользователями для вызова отказа в обслуживании (DoS), которые могут выполнять условия, создающие в ядре бесконечный цикл.

  • CVE-2008-1615

    Ян Кратохвил сообщил о локальном отказе в обслуживании, позволяющем локальным пользователям в системах с ядром для архитектуры amd64 вызывать аварийную остановку системы.

  • CVE-2008-2136

    Пол Харкс обнаружил утечку памяти в коде Simple Internet Transition (SIT), используемом для реализации IPv6 по IPv4-туннелям. Эта уязвимость может использоваться удалёнными пользователями для вызова отказа в обслуживании.

  • CVE-2008-2137

    Дэвид Миллер и Ян Лисковский обнаружили проблемы с проверкой виртуального адресного пространства областей mmaped на архитектуре sparc, что может использоваться локальными пользователями для вызова отказа в обслуживании.

В стабильном выпуске (etch) эта проблема была исправлена в версии 2.6.18.dfsg.1-18etch5.

Сборки для linux-2.6/s390 и fai-kernels/powerpc пока не готовы. Данная рекомендация будет обновлена как только будут доступны указанные сборки.

Рекомендуется обновить пакеты linux-2.6, fai-kernels и user-mode-linux.

Исправлено в:

Debian GNU/Linux 4.0 (etch)
