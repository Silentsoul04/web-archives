Exemplo de auditoria automatizada: pscan

O pscan é um pacote projetado para auditar arquivos fonte C e C++ em busca por vulnerabilidades de formato de string.

Não é uma ferramenta de auditoria de uso geral.

Executando o pscan

Executar pscan é uma simples questão de executá-lo com o nome de um arquivo, ou arquivos, para verificar. Por exemplo:

pscan test.c

A saída será escrita diretamente no console:

test.c:42 SECURITY: printf call should have "%s" as argument 0

A saída

A saída neste caso é fácil de entender. É corretamente identificado o fato que a chamada printf não responde seus argumentos de forma apropriada.

A saída também nos mostra o que devemos fazer para corrigir a falha, altere o código que lê: 

printf( buff );

para ler:

printf( "%s", buff );

Não fazer isso poderia permitir que um(a) invasor(a), que possa controlar a saída de ls, atacar o programa, criando um arquivo chamado "%s", ou semelhante.

Ataques de formato de string são discutidos nesta introdução do foco de segurança.

O HOWTO de programação segurança para Linux e Unix explica como se proteger contra esses ataques em funções variáveis normalmente usadas, como:

  • printf
  • fprintf
  • syslog
