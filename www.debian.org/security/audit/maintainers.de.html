Durchführung eines Audits als Paketbetreuer

Wenn Sie Betreuer eines Pakets sind, das im Debian-Archiv enthalten ist, sollten Sie in Erwägung ziehen, selber den Code durchzusehen.

Die Verfügbarkeit von Werkzeugen, die den Quelltext einem Audit unterziehen, kann diesen Prozess signifikant vereinfachen. Auch wenn Sie keine Zeit haben, ein gründliches Audit selbst durchzuführen, können Sie Bereiche finden, die möglicherweise problematisch sind.

Wenn Sie Hilfe benötigen, kontaktieren Sie bitte entweder das Debian-Sicherheitsteam oder die (öffentliche) debian-security Mailingliste, um Unterstützung dabei zu bekommen, wie Sie ein Quellcode-Audit durchführen.

Informationsquellen für Betreuer

Betreuer, die ihren Quellcode überprüfen möchten, könnten daran interessiert sein, das Debdonf6-Dokument Weeding out security bugs in Debian (Folien) oder die Hinweise Short, practical overview on how to find a few common mistakes in programs written in various languages zu lesen (beide geschrieben von Mitgliedern des Audit-Projekts).

Die Weeding out security bugs in Debian-Präsentation stammt von der Debconf6 in Mexiko und war Teil eines Workshops. Für Betreuer, für die Quellcode-Überprüfung Neuland ist, könnten die Code-Beispiele und die Workshop-Videos nützlich sein.

Neue Veröffentlichungen

Um ein verantwortungsbewusster Betreuer zu sein, sollten Sie neue Veröffentlichungen vom ursprünglichen Autor Ihres Pakets im Auge behalten. Falls das Changelog ein Sicherheitsproblem erwähnt, sollten Sie überprüfen, ob Sie eine Version des Codes in der Stable-Distribution haben, die verwundbar ist.

Wenn Sie eine verwundbare Version in der Stable-Distribution haben, kontaktieren Sie bitte das Sicherheitsteam - wie in der FAQ des Sicherheitsteams beschrieben.
