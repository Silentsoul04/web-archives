Bulletin d'alerte Debian

DSA-068-1 openldap -- Déni de service distant

Date du rapport :
9 août 2001
Paquets concernés :
openldap
Vulnérabilité :
Oui
Références dans la base de données de sécurité :
Dans la base de données de suivi des bogues (chez SecurityFocus) : Identifiant BugTraq 3049.
Dans le dictionnaire CVE du Mitre : CVE-2001-0977.
Les annonces de vulnérabilité et les bulletins d'alerte du CERT : CA-2001-18.
Plus de précisions :
L'annonce CERT liste un nombre de failles de sécurité dans divers implémentations LDAP, basées sur les résultats des séries de test de PROTOS LDAPv3. Ces tests ont trouvé un problème dans OpenLDAP, une implémentation libre LDAP qui est fourni dans Debian GNU/Linux 2.2.

Le problème est que slapd ne gère pas les paquets qui ont les champs BER de longueur invalide et plante s'il les reçoit. Un attaquant pouvait l'utiliser pour réaliser une attaque distante par un déni de service.

Ce problème a été corrigé dans la version 1.2.12-1 et nous vous recommandons de mettre à jour votre paquet slapd immédiatement.

Debian GNU/Linux 2.2 (potato)
